- git reset HEAD filename will take the file out of the commit list/Unstages file changes in the staging area.

- git checkout HEAD filename: will get back before last commit/ Discards changes in the working directory.

- git diff: will show differences between working directory and project

- git status: actual files status (e.g. shows each file: commited untracked etc.)

- git log: shows info about past commits (author etc.)

- git reset SHA: getting back to a specific commit with the give the SHA code

- git branch branch_name: creates a new branch

- git branch: shows all the branches (* <- shows which branch you are currently on)

- git checkout branch_name: switchin to branch_name

- git merge branch_name: merge branch_name into the current branch

- git branch -d branch_name: delete branch named branch_name

- git clone remote_location clone_name: clone the repo @remote_location (cna be either a filepath or a URL) into the folder named clone_name on my machine

- origin: name of the path or url where the repo was cloned from

- git merge: fetch latest changes to origin repo, they will go to a new branch called origin/master (different from master)

- git push origin your_branch_name: push local branch up to the remote(origin)

- git remote show origin: show info about remote and the it's relation to current branch 
e.g. output:
* remote origin
  Fetch URL: https://github.com/robertodessi/openCV.git
  Push  URL: https://github.com/robertodessi/openCV.git
  HEAD branch: master
  Remote branch:
    master tracked
  Local branch configured for 'git pull':
    master merges with remote master
  Local ref configured for 'git push':
    master pushes to master (up to date)

- git push -u origin master: pushes local master to origin repo
